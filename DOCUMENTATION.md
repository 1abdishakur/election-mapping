# Election Monitoring Dashboard - Technical Documentation

## 1. Project Overview
The **Election Monitoring Dashboard** is a state-of-the-art, high-performance web application designed for real-time visualization of national election results. It combines geographical mapping with advanced statistical analytics to provide a single-pane overview of election health, voter turnout, and candidate performance.

## 2. Technology Stack
*   **Core:** HTML5, CSS3, Vanilla JavaScript (ES6 Modules).
*   **Mapping:** [Leaflet.js](https://leafletjs.com/) with custom tile layers, heatmaps, and spatial clustering.
*   **Charts:** [Chart.js](https://www.chartjs.org/) for doughnut and bar visualizations.
*   **Data Ingestion:** Dynamic CSV fetching from Google Sheets (via GViz API).
*   **Styling:** Modern "Glassmorphism" UI with CSS variables and a responsive, flex-based desktop shell.

## 4. Data Architecture & Relational Model
The application uses a **Relational Join Engine** (implemented in `js/dataJoiner.js`) that merges multiple spreadsheet tables into a unified "District Master" object at runtime.

### **Core Relational Schema:**
1.  **States:** Mapping of state names to unique `state_code`.
2.  **Districts:** Master list of districts linked by `dist_code`.
3.  **Party Results:** Vote counts and seats won per party, localized to each district.
4.  **Elected Candidates:** Individual winner details including seat numbers and party logos.
5.  **Centers:** Geospatial coordinates for Registration and Polling centers.

### **Data Normalization:**
To handle real-world spreadsheet inconsistencies, the `DataJoiner` includes:
*   **Case-Insensitive Headers:** All CSV columns are normalized to lowercase.
*   **Smart Matching:** Cross-table joining supports numeric-to-string conversion (e.g., matching `01` to `1`).
*   **Typo Tolerance:** Automatically handles common spreadsheet typos (e.g., `cadidates` mapped to `candidates`).

## 5. Module Breakdown

### **Mapping Engine (`map.js`)**
*   **Choropleth Views:** Dynamic coloring based on Turnout, Winner, or Margin of Victory.
*   **Heatmaps:** Visualizes polling center density across the country.
*   **Coverage Range:** Renders 1km "walking distance" service areas around centers.
*   **Smart Zoom:** Fit-to-bounds logic for state and district-level exploration.

### **Analytics Engine (`charts.js`)**
*   **Vote/Seat Share:** Responsive doughnut charts with automatic scaling.
*   **Parliament Layout:** A custom DOM-based renderer that generates concentric semi-circles representing council seats.
*   **Competitiveness Bar:** Vivid tracking of the "Tightest Races" across the nation.

### **Coordination Layer (`main.js`)**
*   Manages `AppState` (selected state, district, or party).
*   Coordinates state changes between the Filter controls, Sidebar selection, and Map interaction.
*   Handles "Global vs. Specific" context switching.

## 7. Performance & Optimization
*   **Lazy Rendering:** Maps and charts only update when necessary (on data change or resize).
*   **Memoization:** Global summary totals are cached in memory after the initial join.
*   **Desktop-First:** All responsive CSS for mobile has been removed to ensure a stable, no-scroll monitoring experience on Full HD (1920x1080) displays.
